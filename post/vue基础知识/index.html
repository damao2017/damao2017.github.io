1. 核心概念

双向数据绑定：数据变化，页面上用数据的地方会自动变化

通过 指令 扩展HTML，通过 表达式 绑定数据到HTML

2.VUE的引入方式

  1. 本地文件引入

    下载vue.js文件，然后html使用<script>引入

  2. CDN方式引入

    <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>

  3. npm安装

    npm install vue

4.VUE选项 el

<div id="app">
</div>
var app = new Vue({
    el: '#app'
})
  • el指定当前的vue管理的视图
  • 可以是id选择器#app，也可以是class选择器.app
  • 也可以是dom对象document.getElementById(‘app’)
  • 不可以挂在body、html标签

5.VUE选项 data

data里面写数据，指定vue管理的视图中需要用到的数据。在视图中可以通过{{}}访问，还可以通过vue实例.名称来调用。

 var app = new Vue({
            el: '#app',
            data: {
                message: 'Hello Vue!'
            }
        })
//观察app里面的data内容
console.log(app)
//通过vue实例名称.数据名称 获取到内容
console.log(app.message)

6.VUE选项 methods

methods里面写方法。

  1. 调用方法有三种，{{fun()}}，vue.fun()，通过vue指令调用@onclick="fun()”
  2. 方法参数的this指向new Vue()对象，不推荐用箭头函数，没有this
  3. 可以调用data里面的数据
    <div id="app">
        <!-- 调用方法1 -->
        {{hello()}}
        <!-- 调用方法3 -->
        <button @click="hello()">按钮</button>
    </div>
    
    var app = new Vue({
            el: '#app',
            methods:{
                hello(){
                    console.log("hello ---");
                }
            }
        })
        //调用方法2
        app.hello()

8.指令

指令用于增强html标签功能，都是以v-开头，取代之前的DOM操作。

9.指令 v-text和v-html

取代innerText和InnerHtml

v-text替换标签的全部内容，区别于插值表达式{{}}，插值只替换出现他的位置的值，v-text的值来源于data

v-html也可以替换标签的全部内容，并且识别html标签并渲染，存在安全问题，不使用。

<div id="app">
v-text <br>
<span v-text="txt1"></span> <br>
v-html <br> 
<span v-html="txt1"></span> <br>
</div>

var app = new Vue({
	el: '#app',
	data: {
		txt1:"<h4>aaaa</h4>"
		}
})

10.指令 v-on

绑定事件监听器。

在普通元素上时，只能监听原生 DOM 事件；用在自定义元素组件上时，也可以监听子组件触发的自定义事件。

在监听原生 DOM 事件时，方法以事件为唯一的参数。如果使用内联语句，语句可以访问一个 $event 属性：v-on:click="handle('ok', $event)"。

<div id="app">
    v-on <br>
    {{count1}} <br>
    <!-- 函数只有一句执行语句，可以直接写在后面 -->
    <button v-on:click="count1++">v-on1</button> <br>
    <button v-on:click="fn1()">v-on2</button> <br>
    <!-- 函数没有参数，()可以省略 -->
    <button v-on:click="fn1">v-on3</button> <br>
    <!-- v-on:可以简写成@ -->
    <button @click="fn1">v-on4</button> <br>
    <!-- 获取点击事件 -->
    <button @click="fn11(111,$event)">v-on5</button> <br>
    <!-- 监听事件修饰符，详见vue api文档 
    prevent 阻止默认行为-->
    <button @click.prevent="fn1">v-on6</button> <br>
</div>
<script>
    var app = new Vue({
        el: '#app',
        data: {
            message: 'Hello Vue!',
            txt0:"<h4>aaaa</h4>",
            count1 : 1
        },
        methods:{
            hello(){
                console.log("hello ---");
            },
            fn1(){
                this.count1++; 
            },
            fn11(a,e){
                console.log(a);
                console.log(e);   
            }
        }
    })
</script>

11.指令 v-for

基于源数据多次渲染。可以渲染 Array | Object | number | string | Iterable (2.6 新增)

选项:key可以让vue渲染效率更高，必须全局唯一值

<div id="app">
2.v-for <br>
     数组 <br>
     <ul>
         <li v-for="item in listArray2">{{item}}</li>
     </ul>
     <ul>
         <li v-for="(item,index) in listArray2">{{item}}--{{index}}</li>
     </ul>
     对象 <br>
     <ul>
         <li v-for="(value,key,index) in person">{{value}}--{{key}}---{{index}}</li>
     </ul>
     :key <br>
     <ul>
         <li v-for="(value,key,index) in person" :key="index">{{value}}-{{key}}--{{index}}</li>
     </ul>
</div>
<script>
var app = new Vue({
    el: '#app',
    data: {
        listArray2: ['香蕉', '苹果'],
        person2: {
            name: '小明',
            age: 12
        }
    }
</script>

14.指令 v-cloak

可以隐藏未编译的 Mustache 标签直到VUE实例化完成。vue实例化前，网页会显示{{}}未编译的内容。使用脚手架vueloader的避免这种情况发生。

<style>
    [v-cloak] {
        display : none;
    }
</style>

<div id="app" v-cloak>
    {{message}}
</div>

<script>
    setTimeout(() => {
        new Vue({
            el: '#app',
            data: {
                message: 'Hello Vue!',
            }
        })
    }, 2000)
</script>

15.指令 v-once

页面元素只会被渲染一次，以后修改绑定的值，页面不会改变。

16.过滤器

用于一些常见的文本格式化。过滤器可以用在两个地方：双花括号插值和 v-bind 表达式

其实就是一个函数，默认传递参数是管道符前面的那个被处理的数据。

 <!-- 在双花括号中 --> 
{{ message | capitalize }} 

<!-- 在 v-bind 中 -->
 <div v-bind:id="rawId | formatId"></div> 
 
 //全局过滤器
 Vue.filter('capitalize', function (value) {
  if (!value) return ''
  value = value.toString()
  return value.charAt(0).toUpperCase() + value.slice(1)
})
 
 new Vue({
 //局部过滤器
      filters: {
      capitalize: function (value) {
        if (!value) return ''
        value = value.toString()
        return value.charAt(0).toUpperCase() + value.slice(1)
      }
    }
})
//过滤器串联，后面的过滤器处理的是前面的过滤器return的结果
{{ message | filterA | filterB }}
//过滤器接收函数，filterA()就有三个参数，message,a,b
{{ message | filterA('arg1', arg2) }}

Vue.directive('focus', {

19.VUE选项 computed

计算属性:当数据的逻辑很复杂时，可以把对这个数据的逻辑写在计算属性中。

如果data里面a依赖了b，那么data里面写b，然后把a写在计算属性中。a的用法和b是一样的。

区别于选项methods!

<div id="app">
 	7.计算属性 <br>
    原始值：{{text7}} <br>
    计算属性：{{text7com}} <br>
    计算属性：{{text7com}} <br>
    对比methods:{{text7fun()}} <br>
    对比methods:{{text7fun()}} <br>
    <input type="text" v-model="text7"> <br>
</div>
<script>
var app = new Vue({
    el: '#app',
    data: {
        text7: ""
    },
    //多个地方调用，只计算一次
    computed: {
       text7com() {
           console.log("计算属性执行---");
           return this.text7 + "++计算属性添加" + new Date();
       }
    },
    //多个地方调用，执行多次
    methods:{
        text7fun() {
            console.log("函数执行---");
            return this.text7 + "++函数添加" + new Date();
        },
    },
</script>
